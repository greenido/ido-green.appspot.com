Now us...

Why are developers stuck?

  • Cognitive Overload
  • Clientside cost
  • Aversion to learning

Cognitive overload.

Currently the onus is on developers to amass and absorb a huge amount of knowledge in order to be effective front-end developers.

Browser support, progressive enhancement, fallback strategies, edge browser bugs, DOM performance, network performance, best tool for the job..

Clientside cost.

All frontend code is a bandwidth cost passed on to users.

Thus, developers have an excuse for not adding client-side code to their stack.

Every robust solution is "bloated"

Aversion to learning.

Because of the clientside cost aspect, frontend developers stick to the tools they know.

They're out of practice on ramping up on new tools; it's not engrained in their daily regimen.

No developer wants to be spinning their wheels, or operating on mundane tasks when they could instead be creating rich and compelling experiences.

We can arm developers with smart tools, effective workflows, and wise strategies for development.

What are the goals?

Learning and adopting tools are commonplace practice

Defeat the "bloat" mindset

Existing tools gain adoption

New tools solve existing developer challenges

Component Model and MDV get sufficient exposure

Learning and adopting tools are commonplace practice

  • Evangelize the message: don’t be scared of learning; you don’t have to start from scratch.
  • Encourage to hack on working demos.

Defeat the "bloat" mindset

  • Launch onelessjpg.org - which shows that using one less jpg (or image compression) is equal to the cost of adding a “bloated” javascirpt library
  • Champion Build processes and deploy procedures

Existing tools gain adoption

Show that existing tools capture best practices and save developers time in learning browser details. “Outsource esoteric browser detail tracking!”

  • Appcache generation & validation (work with Offline team)
  • Demonstrate power of authoring abstractions like sass/compass, haml, markdown, coffeescript, boostrap
  • Demonstrate power of an IDE or better prototyping tools
  • Onboard more developers to Backbone

New tools solve existing developer challenges

  • HTML5 Boilerplate Plus could present a new, blessed stack with better tools.
  • Modernizr command line interface, builder with community feature detects
  • HTML5 Please to recommend best practices for polyfill implementation

Improve Tools

  • Work with jQuery standards team to improve browsers through standards process
  • Work with jQuery standards team to improve jQuery with new features
  • Get offline (appcache, indexeddb, filesystem) better support in Chrome DevTools

Share Component Model and MDV externally

  • Circulate MDV and Component Model in JS Developer circles.
  • Create a demo, screencast, and post of each.
  • Gather feedback from key developers and liaison to webkit engineers
